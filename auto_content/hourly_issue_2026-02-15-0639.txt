# AI 코딩 시대, 개발자의 "플로우 상태" 지키는 법

## 주제 개요
AI 코딩 도구가 개발자의 업무 방식을 바꾸고 있지만, 많은 개발자들이 AI 도구 사용으로 인해 "플로우 상태(Flow State)"가 깨진다고 느끼고 있습니다. 이 글에서는 AI 코딩 도구를 효과적으로 활용하면서도 개발자의 몰입 상태를 유지하는 방법을 살펴 봅니다.

## 핵심 내용

### 1. AI 코딩의 보이지 않는 비용

AI 코드 생성 도구는 생산성을 높여주지만, 동시에 몰입을 방해하는 요소들도 있습니다:
- **문맥 전환 비용**: AI와 대화하며 코드를 생성하는 과정에서 생각의 흐름이 끊깁니다
- **과도한 의존**: AI 제안에 계속 신경 쓰다 본인의 문제 해결 능력이 퇴화될 수 있습니다
- **불확실성 스트레스**: 생성된 코드의 품질이나 보안성을 계속 검토해야 하는 부담

### 2. 플로우 상태를 지키는 실천 방법

#### (1) AI 사용 구간 명확히 정하기
- **코드 탐색 단계**: AI에게 "이런 기능을 하는 함수가 있을까?" 질문
- **초안 작성 단계**: 반복적인 보일러플레이트 코드는 AI에 위임
- **심화 구현 단계**: 핵심 비즈니스 로직은 직접 작성하며 몰입 유지

#### (2) "AI 톤" 시간과 "딥 워크" 시간 분리
- 하루 중 특정 시간대(예: 오전 2시간)은 AI 도구 완전히 끄고 코딩
- 나머지 시간에는 AI를 활용해 생산성 극대화
- 뇌가 "AI 모드"와 "딥 워크 모드"를 구분할 수 있도록 습관화

#### (3) AI 출력은 "시작점"으로만 사용
- AI가 생성한 코드를 그대로 복사-붙여넣기 하지 않기
- 생성된 코드를 읽고, 이해하고, 자신의 스타일로 재작성하는 과정 거치기
- 이 과정에서 실제 학습과 몰입이 발생합니다

### 3. AI 시대의 새로운 개발자 역량

#### 유지해야 할 역량
- **문제 분해 능력**: 큰 문제를 작은 단위로 쪼개는 능력 (AI에게 정확한 지시를 내리기 위해 필수)
- **코드 리뷰 능력**: AI 생성 코드의 품질을 판단하는 안목
- **아키텍처 설계**: 전체 시스템을 설계하는 고차원적 사고

#### 새롭게 필요한 역량  
- **프롬프트 엔지니어링**: AI에게 효과적으로 요구사항 전달하는 기술
- **AI-휴리스틱 밸런싱**: 언제 AI를 쓰고 언제 직접 코딩할지 판단하는 감각

### 4. 실전 팁: VS Code 설정으로 AI와 건강한 관계 만들기

```json
{
  "github.copilot.enable": {
    "*": true,
    "markdown": false,  // 문서 작성할 땐 AI 끄기
    "plaintext": false
  },
  "github.copilot.inlineSuggest.enable": false  // 자동 제안 끄고, 탭으로만 호출
}
```

### 5. 마무리: AI는 도구일 뿐

AI 코딩 도구는 마치 "자동변속기"와 같습니다. 편리하지만, 수동변속기(직접 코딩)를 완전히 대체할 수는 없습니다. 중요한 건 **상황에 맞게 선택하는 지혜**입니다.

플로우 상태를 유지하면서 AI의 도움을 받으려면:
1. AI에게 "생각"을 맡기지 않고 "반복 작업"만 맡기기
2. 매일 최소 1-2시간은 AI 없이 코딩하는 시간 가지기
3. AI가 생성한 코드는 반드시 이해하고 소화하기

AI 시대에도 여전히 **"생각하는 개발자"**가 되는 것이 중요합니다.
