VS Code 생산성 2배로 만드는 필수 단축키 15가지 + 설정 꿀팁

2026년 2월 19일 개발

왜 단축키가 중요할까?

개발할 때 가장 많이 하는 행동이 뭔지 생각핸지? 코드를 치는 것도 중요하지만, 사실 파일을 찾아다니고, 코드를 수정하고, 여러 창을 왔다갔다 하는 시간이 훨씬 더 많아. 마우스로 메뉴를 클릭하면서 "파일 열기"를 찾고, 다시 마우스를 움직여서 코드 위치로 가고... 이게 하루에 수십 번 반복되잖아.

내가 처음 VS Code를 썼을 때는 단축키 하나도 몰랐어. 그냥 마우스로 다 했거든. 근데 개발자 친구가 옆에서 키보드만 두드리는데 화면이 막 바뀌고 파일이 쑥쑥 열리는 거야. "야 너 뭐 했어?" 물어봤더니 "그냥 단축키"라는 거야. 그때부터 배우기 시작했는데, 진짜 생산성이 달라지더라고.

단축키를 쓰면 손이 키보드에서 안 떠도 되니까 흐름이 끊기지 않아. 마우스로 화면을 클릭하면 집중력이 흐트러지거든. 근데 키보드로 다 하면 코드에만 집중할 수 있어. 통계적으로 단축키를 적극 활용하는 개발자는 그렇지 않은 개발자보다 하루에 30~40분은 더 아낄 수 있다고 해. 한 달이면 10시간 이상이야. 이 시간으로 한 권의 책도 다 읽을 수 있겠지?

파일 탐색 단축키 - 손 안 떼고 파일 열기

1. Ctrl+P (Mac: Cmd+P) - 빠른 파일 열기

이건 진짜 필수 중의 필수야. 파일 탐색기에서 폰데률이면서 찾을 필요 없이, 그냥 이름의 일부만 쳐도 파일이 검색돼. 예를 들어 "index.js"를 찾고 싶으면 Ctrl+P 누르고 "ind"만 쳐도 나와. 대형 프로젝트에서 진짜 시간을 엄청 아껴줘.

꿀팁: 파일명 뒤에 ":"을 붙이면 해당 파일의 특정 줄로 바로 이동해. 예를 들어 "app.js:150"이라고 치면 app.js 파일의 150번째 줄로 바로 가거든.

2. Ctrl+Shift+F (Mac: Cmd+Shift+F) - 전체 검색

프로젝트 전체에서 특정 텍스트를 찾을 때 쓰는 거야. 함수명이나 변수명을 바꿀 때 특히 유용해. 검색어를 입력하면 모든 파일에서 찾아주고, 검색 결과에서 바로 더블클릭해서 해당 위치로 이동할 수도 있어.

꿀팁: 정규식 검색도 지원하거든. 복잡한 패턴 찾을 때 유용해. 예를 들어 "function.*test" 이런 식으로 쓰면 "function"으로 시작하고 "test"가 들어가는 라인을 다 찾아줘.

3. Ctrl+Shift+N (Mac: Cmd+Shift+N) - 새 창 열기

여러 프로젝트를 동시에 작업할 때 쓰는 거야. 새 VS Code 창을 열어서 다른 폴터를 열 수 있어. 예를 들어 프론트엔드와 백엔드를 동시에 작업할 때 하나의 창에서 왔다갔다 하면 헷갈리잖아. 이럴 때 새 창으로 분리하면 깔끔해.

4. Ctrl+W (Mac: Cmd+W) - 현재 탭 닫기

안 쓰는 파일 탭을 빠르게 닫을 수 있어. 마우스로 x 버튼 찾아서 클릭할 필요 없이 그냥 Ctrl+W 누륨돼. 여러 파일을 열어놓고 작업하다 본전 탭이 수십 개 쌓일 때가 있거든. 이때 하나씩 닫기 귀찮으면 Ctrl+K W로 다 닫을 수도 있어.

코드 편집 단축키 - 마우스 없이 코드 수정하기

5. Alt+↑ / Alt+↓ (Mac: Option+↑ / Option+↓) - 라인 이동

이거 진짜 자주 쓰게 돼. 현재 라인을 위아래로 옮길 수 있는 거야. 예를 들어 변수 선언 위치를 바꾸고 싶을 때, 드래그해서 잘라내고 붙여넣기 할 필요 없이 그냥 Alt+방향키로 쑥 옮기면 돼. 여러 줄 선택하고 핸도 되거든.

6. Alt+Shift+↑ / Alt+Shift+↓ (Mac: Option+Shift+↑ / Option+Shift+↓) - 라인 복제

비슷한 코드를 반복해서 써야 할 때 유용해. 현재 라인을 그대로 위나 아래로 복사하는 거야. 예를 들어 console.log를 여러 개 찍어야 할 때, 한 줄 쓰고 Alt+Shift+↓ 누륨돼. 드래그해서 복사 붙여넣기 하는 것보다 훨씬 빨라.

7. Ctrl+D (Mac: Cmd+D) - 같은 단어 선택

변수명이나 함수명을 한꺼번에 바꿀 때 쓰는 거야. 예를 들어 "userName"이라는 변수가 코드에 10번 나오는데 다 바꾸고 싶으면, 커서를 "userName" 위에 두고 Ctrl+D를 계속 누륨돼. 누를 때마다 다음 "userName"이 선택되거든. 다 선택했으면 그냥 새 이름을 치면 한꺼번에 다 바뀌어.

주의: 이 기능은 문자열 안에 있는 텍스트도 선택하니까 주의해야 해. 문자열까지 바꾸고 싶지 않으면 Ctrl+Shift+L 대신 쓰면 되거든.

8. Ctrl+Shift+L (Mac: Cmd+Shift+L) - 전체 같은 단어 선택

파일 전체에서 같은 단어를 한꺼번에 선택하는 거야. Ctrl+D는 하나씩 선택하는 거라면, 이건 한 번에 다 선택핲돼. "var"를 모두 "let"으로 바꾸고 싶을 때 유용하지. 근데 조심해야 할 게, 주석이나 문자열 안에 있는 것까지 다 바꿔버리거든.

9. Ctrl+/ (Mac: Cmd+/) - 주석 토글

코드를 잠깐 주석 처리하고 싶을 때 쓰는 거야. 여러 줄을 드래그하고 Ctrl+/를 누륨돼. 다시 누륨사 주석이 풀리거든. 디버깅할 때 특히 유용해. console.log를 찍어놨는데 나중에 다 지우기 귀찮으면 주석 처리핲돼.

10. Ctrl+Shift+K (Mac: Cmd+Shift+K) - 현재 라인 삭제

현재 라인을 통째로 지우는 거야. 커서가 어디 있든 그 라인 전체가 삭제돼. 드래그해서 지우는 것보다 훨씬 빠르지. 특히 불필요한 빈 줄 정리할 때 편해. Ctrl+X로 잘라내기 핸도 되지만, 이건 클립보드에 안 남거든.

화면 조작 단축키 - 멀티 뷰 활용하기

11. Ctrl+\ (Mac: Cmd+\) - 화면 분할

화면을 두 개로 나눠서 다른 파일을 동시에 볼 수 있어. 예를 들어 HTML이랑 CSS를 동시에 볼 때 유용하지. Ctrl+\를 누륨돼. 세 개, 네 개로도 나눌 수 있거든. 다시 원래대로 하고 싶으면 Ctrl+W로 닫음돼.

12. Ctrl+1, Ctrl+2, Ctrl+3... - 분할 화면 이동

화면을 분할핸다면 분할된 화면 사이를 이동할 때 쓰는 거야. Ctrl+1은 첫 번째 화면, Ctrl+2는 두 번째 화면으로 이동하는 거지. 마우스로 클릭하지 않고 키보드만으로 화면을 옮길 수 있어서 흐름이 끊기지 않아.

13. Ctrl+B (Mac: Cmd+B) - 사이드바 토글

왼쪽 파일 탐색기를 보였다 안 보였다 하는 거야. 화면이 좁을 때 코드만 보고 싶으면 Ctrl+B로 사이드바를 숨겨. 다시 필요하면 똑같이 누륨돼. 화면 공간을 효율적으로 쓸 수 있어서 노트북에서 작업할 때 특히 유용해.

14. Ctrl+J (Mac: Cmd+J) - 터미널 토글

VS Code 내장 터미널을 열고 닫는 거야. 별도의 터미널 창을 띄울 필요 없이 그냥 Ctrl+J로 바로 켤 수 있거든. npm install이나 git 명령어 칠 때 매우 편해. 다 썼으면 다시 Ctrl+J로 숨기고 코드로 돌아가면 돼.

15. F12 - 정의로 이동

함수나 변수를 정의한 곳으로 바로 이동하는 거야. 예를 들어 "calculateTotal()"이라는 함수를 호출하는 곳에서 F12를 누륨돼. 그 함수가 어디서 정의됐는지 바로 찾아가줘. 다시 원래 위치로 오고 싶으면 Alt+←를 누륨돼. 코드를 따라가며 이해할 때 필수야.

추천 settings.json 설정

단축키만큼 중요한 게 설정이야. VS Code의 설정 파일인 settings.json에 몇 가지를 추가하면 개발 경험이 훨씬 좋아져. 파일 → 기본 설정 → 설정 JSON 열기에서 수정할 수 있어.

{
  "editor.fontSize": 14,
  "editor.lineHeight": 24,
  "editor.tabSize": 2,
  "editor.insertSpaces": true,
  "editor.wordWrap": "on",
  "editor.minimap.enabled": false,
  "editor.formatOnSave": true,
  "editor.bracketPairColorization.enabled": true,
  "editor.guides.bracketPairs": true,
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  "terminal.integrated.defaultProfile.windows": "PowerShell",
  "terminal.integrated.defaultProfile.osx": "zsh",
  "workbench.startupEditor": "none",
  "explorer.confirmDelete": false,
  "explorer.confirmDragAndDrop": false
}

이 설정들을 하나씩 설명해줄게. fontSize와 lineHeight는 눈의 피로를 줄여줘. tabSize를 2로 하면 들여쓰기가 짧아져서 화면에 더 많은 코드를 볼 수 있어. wordWrap을 켜면 가로로 긴 코드가 자동으로 줄바꿈돼서 가로 스크롤 안 핸돼. minimap은 오른쪽에 있는 미니맵인데, 나는 끄는 편이야. 화면을 더 많이 쓸 수 있거든.

formatOnSave는 파일을 저장할 때 자동으로 코드 포맷을 정리해주는 거야. Prettier 같은 확장 프로그램이 있으면 그 규칙에 맞춰서 정리해줘. bracketPairColorization은 괄호 쌍을 다른 색으로 표시해줘. 중첩된 괄호가 많을 때 헷갈리지 않게 도와줘. autoSave는 파일을 자동으로 저장해주는 거야. Ctrl+S 누르는 걸 잊어버려서 날린 적 있지? 그런 실수를 방지해줘.

자주 하는 실수와 해결법

단축키를 처음 배울 때 몇 가지 실수를 하기 쉬워. 첫째, 너무 많은 단축키를 한꺼번에 외우려고 하는 거야. 15개를 하루에 다 외우려 하면 금방 지치거든. 그냥 매일 한두 개씩 익혀. 일주일이면 다 익힐 수 있어.

둘째, 단축키를 쓸 때마다 의식적으로 생각하는 거야. 처음에는 당연히 그럴 수밖에 없지. 근데 반복해서 쓰다 본전 손이 기억하게 돼. 그때까지는 의식적으로라도 써봐. 안 쓰면 영영 못 외워.

셋째, 내가 쓰는 운영체제의 단축키를 제대로 안 보는 거야. Windows랑 Mac이 다르거든. 이 글에서는 둘 다 표기했지만, 혼동하지 않게 주의해야 해. 회사에서는 Windows 쓰다가 집에서 Mac 쓰면 헷갈릴 때가 있어. 그럴 땐 Keybindings 설정에서 단축키를 통일시킬 수도 있거든.

마무리 - 오늘부터 실천하기

오늘 소개한 단축키 15가지와 설정 팁을 잘 활용하면 개발 속도가 확실히 빨라질 거야. 하지만 제일 중요한 건 꾸준히 쓰는 거야. 오늘 배운 것 중에서 지금 당장 쓸 수 있는 것부터 시작핸지. 예를 들어 Ctrl+P로 파일 열기부터 익숙해지는 거지.

한 달 정도 꾸준히 쓰면 이제 단축키 없이는 못 사는 몸이 돼 있을 거야. 마우스를 향해 손이 가려다가 키보드로 돌아오는 순간이 올 거야. 그때가 되면 진짜 개발자로 한 단계 성장한 거라고 복면 돼.

추가로 궁금한 단축키나 팁이 있으면 댓글로 물어봐줘. 더 많은 VS Code 팁들을 공유해줄게. 그럼 오늘도 행복한 코딩 되길 바랄게! 다음에는 Git을 VS Code에서 효율적으로 쓰는 방법에 대해 글을 써볼까 하는데, 관심 있으면 알려줘.
